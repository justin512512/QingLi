# 轻历信息型万年历重设计规格

## 1. 目标

将轻历从“系统日历的替代外观”升级为离线优先的信息型万年历。用户单击 Windows 11 任务栏右下角的时间和日期区域，即可打开一个紧凑三栏窗口，集中查看历史上的今天、农历与传统黄历、月历、节气节日倒计时、生日和纪念日。

成功标准：

- 首次运行后，时间和日期区域由轻历安全接管，整块区域可单击。
- 单击后 200 毫秒内显示日历窗口；再次单击或点击窗口外部关闭。
- 主窗口约 900×440，在常见 Windows 11 缩放比例下保持三栏且不超出屏幕。
- 截图参考中的主要信息均在首页直接可见，而不是隐藏在托盘菜单或设置页。
- 核心信息离线可用，无账号、会员或付费功能。

## 2. 技术边界

Windows 没有公开、受支持的接口允许第三方替换系统时钟原生的日历弹窗。轻历采用等效的安全接管方式：保存并隐藏原生时钟，在完全相同的位置显示轻历自己的时间日期窗口，并由该窗口处理单击后打开三栏日历。

明确不采用：

- 不向 Explorer 注入 DLL。
- 不修改 Windows 系统文件。
- 不使用未公开的 Shell 内部接口劫持原生日历。
- 不依赖隐藏托盘图标作为主要入口。

## 3. 窗口与视觉结构

### 3.1 任务栏时间区域

- 占用原时间日期区域的完整矩形。
- 第一行显示当前时间，第二行显示公历日期；外观跟随任务栏深浅色。
- 整块区域是一个点击目标，不要求用户点击隐藏托盘图标。
- 单击切换日历窗口显示状态。

### 3.2 紧凑三栏窗口

窗口目标尺寸约 900×440，靠近被点击的任务栏时间区域显示，并根据工作区边界自动调整位置。

左栏“历史上的今天”：

- 显示所选月日的历史事件。
- 每项包含年份和简短事件说明。
- 内容超出时仅左栏滚动。

中栏“日期与月历”：

- 左侧显示星期、大号日期、农历月日、干支年/月/日和生肖。
- 显示“宜”和“忌”，底部注明“传统黄历参考”。
- 右侧显示年月切换、今天按钮、七列六行月历。
- 周末使用红色；非本月日期弱化；所选日使用淡蓝底；节气和节日使用蓝色；法定休息日和调休工作日显示“休/班”角标。

右栏“最近事件”：

- 合并未来 90 天的节气、传统节日、法定节假日、生日和纪念日。
- 按距离天数升序排列，显示日期和“还有 N 天”。
- 蓝色代表节气，红色代表节日/假日，紫色代表生日，橙色代表纪念日。
- 内容超出时仅右栏滚动。

底部快捷入口：

- 添加生日。
- 添加纪念日。
- 设置。

## 4. 操作行为

- 点击月历日期后，三栏同步切换到该日期：左栏加载对应历史事件，中间更新日期详情和宜忌，右栏以该日期为起点重算未来 90 天。
- “今天”按钮恢复当前日期和当前月份。
- 点击右栏节气或节日，月历选中对应日期。
- 点击生日或纪念日，打开查看/编辑窗口。
- 添加生日支持公历或农历、出生年份、备注、每年重复和提前提醒天数。
- 添加纪念日支持起始日期、起始年份、备注、每年重复和提前提醒天数。
- 生日显示即将到来的年龄；纪念日显示即将到来的周年数。
- 日历窗口出现后不抢占正在输入的应用；用户点击其他区域后关闭。

## 5. 离线数据设计

### 5.1 农历与传统黄历

采用 MIT 许可的 `lunar-csharp` 1.6.8，并固定版本。该库离线计算农历、干支生肖、节气、传统节日和每日宜忌。轻历通过自己的领域接口封装第三方库，界面层不直接依赖其类型。

项目来源：<https://github.com/6tail/lunar-csharp>

### 5.2 历史上的今天

- 内置 `history-today.zh-CN.json`，覆盖 366 个 MM-dd 键，每日最多显示 10 项。
- 每项保存年份、简短事实、来源名称和来源链接。
- 初始数据包作为应用资源随包发布，运行时不需要网络。
- 更新包必须带版本、生成时间和 SHA-256；校验通过后才替换本地旧包。
- 界面只展示事实性短摘要，不复制来源中的长段落。

### 5.3 节假日与调休

- 延续按年份保存的中国大陆节假日 JSON 数据包。
- 内置当前年度数据；跨到无数据年度时仍显示固定日期节日和计算型传统节日，但明确提示“该年度放假调休数据未更新”。
- 可选更新不要求登录，不影响离线使用。

### 5.4 用户数据

- 生日、纪念日、提醒历史和暂停状态保存在本地 SQLite 数据库。
- 不上传联系人、生日或纪念日。
- 数据库升级使用可回滚迁移；迁移失败时保留原文件并停止写入。

## 6. 组件边界

- `IAlmanacService`：根据公历日期返回农历、干支生肖、宜忌、节气和传统节日。
- `IHistoryTodayProvider`：按月日读取历史事件，不负责网络更新。
- `IHolidayProvider`：读取年度假日、休/班状态和数据版本。
- `IUpcomingEventService`：合并节气、节日、生日和纪念日，计算未来 90 天排序结果。
- `IAnniversaryRepository`：保存和编辑纪念日；生日继续使用现有 `IBirthdayRepository`。
- `CalendarDashboardViewModel`：协调三栏所需数据和日期切换，不包含 Win32 任务栏逻辑。
- `ClockReplacementCoordinator`：继续负责系统时钟快照、显示自定义时钟、隐藏原生时钟和失败回滚。
- `CalendarPopupWindow`：负责紧凑三栏布局、窗口定位和显示/关闭行为。

各服务只通过稳定的数据记录交付界面所需字段，避免第三方日历库、SQLite 或 Win32 类型泄漏到其他层。

## 7. 数据流

1. 用户单击任务栏时间区域。
2. 任务栏时钟窗口调用现有单实例日历切换入口。
3. `CalendarDashboardViewModel` 立即生成当月日期骨架，同时并行读取黄历、历史事件和未来事件。
4. 月历骨架准备后立即显示窗口；未完成的栏显示加载占位，并在各自数据返回后独立更新。
5. 用户选择其他日期时取消上一次未完成加载，只更新最新日期。
6. 用户新增或编辑生日/纪念日后，右栏重新查询并保持当前选中日期。

## 8. 错误处理与恢复

- 系统时钟接管继续遵循“捕获状态→持久化快照→显示轻历时钟→隐藏原生时钟→保存启用状态”的顺序。
- 接管任一步骤失败都恢复系统时钟并隐藏轻历时钟。
- 独立 `QingLi.Recovery.exe` 保留用于紧急恢复。
- 黄历服务失败时仅显示“黄历数据暂不可用”，月历、生日和任务栏时钟继续工作。
- 历史数据缺失或损坏时左栏显示“暂无历史数据”，不影响其他栏。
- 年度放假数据缺失时不显示错误休/班标记，并显示数据未更新提示。
- 更新包先写临时文件，完成格式、版本和 SHA-256 校验后原子替换；任何失败继续使用旧包。
- 管理员账户下不可用的系统通知仅以非阻塞状态提示，不弹出模态警告。

## 9. 性能与兼容性

- 日历窗口首次显示目标不超过 200 毫秒；历史列表可随后异步填充。
- 月份切换和日期选择不得阻塞 UI 线程。
- 缓存当前月、前一月和后一月的日历计算结果。
- 支持 Windows 11 底部任务栏、100%、125%、150%、175% 和 200% 缩放。
- 多显示器时窗口出现在被点击时间区域所在显示器，并限制在该显示器工作区内。
- Explorer 重启、显示设置变化和睡眠唤醒后重新定位轻历时钟；定位失败时恢复原生时钟。

## 10. 测试与人工验收

自动测试覆盖：

- `lunar-csharp` 适配器对固定日期的农历、干支、节气和宜忌映射。
- 366 日历史数据包的模式、索引、排序和损坏回退。
- 年度节假日数据及休/班角标。
- 生日与纪念日的公历/农历重复、年龄/周年数和提前提醒。
- 未来 90 天合并、排序、同日多事件和跨年行为。
- 三栏日期同步、快速切换取消和错误状态。
- 任务栏时钟单击切换、单实例激活、窗口贴边和 DPI 换算。
- 接管失败、异常退出、缺少快照、损坏快照和恢复工具。

人工验收覆盖：

- 单击任务栏时间日期整块区域打开，第二次单击关闭。
- 三栏显示完整信息且各栏滚动互不干扰。
- 新增、编辑和删除生日/纪念日后，右栏立即更新。
- 断网时所有核心信息可用。
- 任务栏缩放、多显示器、Explorer 重启、睡眠唤醒和程序异常终止后的恢复。

## 11. 交付范围

本轮交付包含紧凑三栏界面、离线黄历与历史数据、节日节气倒计时、纪念日、生日强化、任务栏整块点击和安全恢复。

本轮不包含天气、云同步、账号体系、跨设备同步、付费会员或直接修改 Windows 原生日历。
