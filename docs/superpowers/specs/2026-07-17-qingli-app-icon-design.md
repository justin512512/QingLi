# 轻历应用图标设计

## 目标

为轻历建立统一且易识别的 Windows 应用图标，替换当前由系统生成的默认程序图标。图标需要同时适用于桌面快捷方式、任务栏、托盘、窗口标题栏和发布包。

## 视觉方案

采用用户确认的 A 方案“日历＋月牙”，不使用汉字或拉丁文字。

- 主体为白色日历页，顶部使用深蓝色装订栏。
- 右下角放置金黄色月牙，表达农历、节气与夜间时钟。
- 背景使用蓝青色至靛蓝色的柔和渐变，与轻历界面现有蓝色强调色协调。
- 外轮廓采用 Windows 11 风格的圆角方形。
- 造型保持扁平、清晰，不使用照片质感、复杂纹理、文字、水印或细碎装饰。

## 尺寸适配

交付透明背景 PNG 源图和包含标准 Windows 尺寸的 ICO 文件。ICO 至少包含 16、20、24、32、40、48、64、128 和 256 像素图层。

- 48 像素及以上保留装订栏、日历页和月牙的完整层次。
- 32 像素简化装订细节并增强轮廓对比。
- 16 至 24 像素只保留蓝色圆角底、白色日历块和黄色月牙三个主要色块，避免缩小后模糊。
- 所有尺寸必须保留安全边距，不让主体贴近画布边缘。

## 程序集成

- 将 ICO 作为 `QingLi.Windows` 的应用程序图标和可执行文件图标。
- WPF 主窗口继承应用图标，避免标题栏继续显示默认图标。
- 托盘图标读取同一图标资源，不再使用运行时绘制的占位符。
- 恢复工具继续保持独立用途，但可复用同一品牌图标。
- 发布版本升级为 v0.1.2，并重新生成 Windows x64 便携包。

## 验收标准

- 桌面快捷方式不再显示默认白色窗口图标。
- 任务栏、隐藏图标区、窗口标题栏和资源管理器中的图标一致。
- 16×16 与 32×32 尺寸可清楚辨认日历与月牙轮廓。
- 深色和浅色桌面背景下均有足够对比度。
- 项目完整测试通过，发布包中的 EXE 带有新图标。
