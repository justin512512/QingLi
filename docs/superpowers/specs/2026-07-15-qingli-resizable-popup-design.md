# 轻历日历弹窗自由布局设计

## 目标

将当前固定位置、固定尺寸的无边框日历弹窗改为可移动、可自由缩放且能记住布局的桌面窗口，同时保留圆角三栏界面和任务栏时钟点击打开行为。

## 交互设计

- 弹窗默认尺寸为 `1040 × 520`，最小尺寸为 `760 × 420`。
- 保留无边框和圆角外观，不增加 Windows 标准标题栏。
- 窗口顶部提供透明拖动区；按住拖动区可移动窗口。
- 窗口四边和四角支持系统级缩放，右下角显示轻量缩放提示。
- 首次打开时仍根据任务栏点击位置自动靠近任务栏显示。
- 用户完成移动或缩放后，保存窗口的左、上、宽、高。
- 后续打开优先恢复用户布局，不再每次强制回到任务栏旁。
- 设置页增加“恢复默认窗口布局”按钮。执行后清除自定义布局，并在下次显示时恢复默认尺寸和自动定位。

## 布局保存

新增独立的本地窗口布局存储，保存于轻历数据目录，不与生日、纪念日数据库混合。布局字段包括：

- `left`
- `top`
- `width`
- `height`
- `isCustomized`

保存采用临时文件替换方式，避免异常退出留下半写入文件。读取失败或数据不合法时忽略该文件并使用默认布局。

## 多显示器与安全恢复

- 恢复布局时根据保存矩形与当前各显示器工作区的相交面积选择目标显示器。
- 如果原显示器已移除，使用鼠标所在显示器。
- 窗口至少保留一块可拖动区域在工作区内；完全越界时自动约束回目标工作区。
- 当保存尺寸大于当前工作区时，按最小尺寸和工作区上限进行收缩。
- DPI 变化后使用 WPF 设备无关单位重新约束，不保存物理像素坐标。

## 组件调整

- `CalendarPopupWindow`：处理拖动、缩放完成事件、布局加载与保存。
- `CalendarPopupLayoutStore`：负责布局文件的读取、校验、原子保存和清除。
- `CalendarPopupPlacement`：扩展为支持保存矩形约束和多显示器回退。
- `SettingsViewModel` / `SettingsWindow`：提供恢复默认布局命令和反馈。

组件之间通过接口传递布局数据，窗口不直接解析 JSON，便于自动测试。

## 错误处理

- 布局文件缺失、损坏、包含非有限数值或尺寸过小时，直接使用默认布局。
- 保存布局失败不阻止日历使用；设置页显示非阻塞错误信息。
- 恢复默认布局失败时保留当前窗口，不删除其他用户数据。

## 验收与测试

- XAML 允许缩放并包含拖动区、最小尺寸和缩放提示。
- 拖动只由顶部拖动区触发，不影响日期、按钮和滚动列表点击。
- 缩放后宽高不会小于 `760 × 420`。
- 移动或缩放完成后保存一次最终布局，避免拖动过程中高频写盘。
- 再次打开恢复保存的位置和尺寸。
- 保存布局在主屏、副屏、显示器移除和 DPI 改变后都能保持可见。
- “恢复默认窗口布局”会清除保存数据，并让下一次打开使用 `1040 × 520` 和任务栏自动定位。
- 现有日历、任务栏时钟、提醒、托盘和打包测试保持通过。

## 不在本次范围

- 不提供任意比例的界面字体缩放设置。
- 不改变三栏信息结构。
- 不将日历长期固定在桌面或设置为普通任务栏窗口。
- 不直接注入或修改 Windows Explorer。
